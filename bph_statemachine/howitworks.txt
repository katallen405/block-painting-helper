Top level:

UI can send messages 
- to change arm mode (teleop, hold position, move to location, springcontroller mode) 
- to request an object (specify the object and then send the mobile manipulator to go get it) 
- To debug why a motion plan failed (for the arm or the turtlebot) and request a new plan
- To show the virtual spring model in springcontroller
- To add or remove springs in springcontroller

Arm can be in one of four modes.  
- In movetolocation, states are "waiting for a location", "planning to a location", "moving to the location", "grasping object", "holding object" and maybe "failedtograsp"?  
- In springcontroller,  the states are running, add spring, remove spring, stopped
- In teleop, the states are running and done (transition with user button)
- In hold position, the state is running and done (transition with user request)

Turtlebot can be in one of two modes:
- waiting
- navigating to location

Gripper can be in one of two modes: 
- open
- closed


Arm States:
ready to pick up object
pre: an object is in the workspace for the arm
pre: we know the location of the object
action: send location of the object to bph_pickmeup
transitions: 
success:  (planned to location, known via feedback from bph_pickmeup action server) go to "moving to pick up object"
failure:  (can not plan to location) go to "move object on mobile base"
failure: (precondition not obtained) go to "find object location"


moving to pick up object
pre: an object is in the workspace for the arm
pre: we have a plan to the location of the object (via feedback from bph_pickmeup)
action: move end effector to location
transitions: success (ee in location of object) go to "pick up object"
failure:  object moved (return to "find object location")
failure: ee did not move to location (return to ready to pick up object)

pick up object
pre: an object is in the workspace for the arm
pre: we are at a gripping location for the object
action: grip the object
success: go to "object in gripper move to workspace"
failure: go to "find object location"

object in gripper move to workspace:
pre: object in gripper
action:  plan to workspace
success:  activate spring controller (go to springcontroller)
failure: go to object in gripper move to workspace, increment failure count
failure: failcount > limit, go to ask user for help

springcontroller:
pre: object in gripper
pre: end effector in workspace
action: activate spring controller
transition:  button to change to position hold mode pressed, go to "position holding"
transition: gripper open requested, go to "open gripper for user"

open gripper for user:
pre: object in gripper
pre: open gripper requested with button
action: when button pressed, open gripper to drop object
transition: waiting mode
